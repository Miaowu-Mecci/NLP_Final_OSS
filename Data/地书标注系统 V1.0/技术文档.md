    # 《地书》图形语言多级分组与多任务标注系统
    
    ## 技术架构与实现文档 (v5.0)
    
    ### 1. 系统概述
    
    本系统是一个纯前端（Vanilla JS + HTML + CSS）实现的数据标注工具，专门为《地书》图形语言序列的多层级结构化和多维度语义标注而设计。
    
    系统针对自动切分后的“最小图形单元（Atom）”进行操作，主要提供两大核心能力：
    
    - **Grouping（多级分组）**：支持跨单元、跨层级的连续选区构建与嵌套组合。
    - **Labeling（多任务标注）**：支持通过动态表单对任意层级（Atom或Group）进行语义、属性、量表等多维度录入。
    
    ### 2. 技术栈与文件结构
    
    - **技术栈**：HTML5, CSS3 (Flexbox/Grid), 原生 JavaScript (ES6+)。无第三方框架依赖。
    - **文件目录结构**：
        - `index.html`：DOM骨架，包含登录屏与主工作台两部分。
        - `style.css`：样式表，负责响应式布局、悬浮组件、交互高亮以及防页面断裂的滚动流设计。
        - `app.js`：核心业务逻辑控制器，处理状态管理、DOM渲染、事件派发和数据持久化。
        - `data/segments_manifest.js`：底层原子图形数据清单（输入）。
        - `data/config.js`：标注任务的动态配置文件（输入）。
    
    ### 3. 核心数据模型 (Data Models)
    
    系统在内存中维护一套严格的数据模型，确保视图与数据的解耦。
    
    ### 3.1 原子图形 (Atom Segment)
    
    基于 `segments_manifest.js` 归一化后的数据。
    
    - `id`: 唯一标识 (例如 `seg_000001`)
    - `global_order`: 绝对排序号，用于判断对象间的物理连续性。
    - `page_no`, `side`: 页码与侧边（L/R），系统将其联合构成为独立的物理页 Key（如 `59_L`）。
    - `line_no`: 行号。
    
    ### 3.2 分组对象 (Group)
    
    在 `state.groups` 中维护的用户创建的分组。
    
    - `id`: 唯一标识 (如 `grp_123456_78`)
    - `level`: 所在层级，基于子元素的最高层级+1。
    - `children`: 包含的子节点 ID 列表（Atom ID 或 Group ID）。
    - `leaf_start` / `leaf_end`: 该组所覆盖的底层 Atom 的起止 `global_order`，用于极速判断选区重叠。
    
    ### 3.3 标注数据 (Annotation)
    
    在 `state.annotations` 中维护的任务结果。
    
    - `target_id`: 被标注对象的 ID（Atom 或 Group）。
    - `task_id`: 对应的标注任务 ID。
    - `value`: 标注值（根据任务类型可为 String, Array, 或 Object）。
    - `updated_at`: 更新时间戳。
    
    ### 4. 核心交互与业务逻辑
    
    ### 4.1 独立分页与工作区过滤
    
    由于存在同一页码的左右分版（L/R），系统在启动时扫描数据源，自动生成基于 `[页码_侧边]`（如 `59_L`）的联合 Key 作为独立的物理页面。用户在登录界面选择工作范围后，系统会过滤并仅渲染落在该范围内的序列数据。
    
    ### 4.2 Grouping 模式（连续多级分组）
    
    - **选区构建**：支持在原子图像上按住鼠标**拖拽划选**，或通过**点击**现有的 Group/Atom 来扩充选区。系统会执行 `isAdjacentOrOverlapping` 算法严格检验跨度连续性。
    - **悬浮工具栏**：当生成有效的连续选区后，DOM会动态计算起始位置，并在上方弹出浮动菜单（包含 `Group` 和 `取消选择` 按钮）。
    - **层级解除 (Ungroup)**：系统计算所有分组，仅为“最高层级（没有被其他父组包含）”的 Group 渲染右上角的红色 `×` 按钮，防止误删破坏层级树。
    
    ### 4.3 Labeling 模式与“静默自动保存”
    
    - **表单动态生成**：读取 `config.js`，支持 Text(单行/多行), Single(单选), Multi(多选), Multi-text(多标签文本), Scale(离散/连续量表) 五种UI组件。
    - **焦点保护与静默更新 (`updateSummaryDOM`)**：所有输入控件绑定 `change` 或 `input` 事件，触发后数据立即写入 state。为防止全量渲染 `rerenderAll()` 导致输入框失去焦点，系统针对文本和滑动条采用**局部DOM更新算法**，仅刷新主工作区对应元素的文本摘要（Summary）和高亮状态，实现无缝连贯的标注体验。
    - **跨级联动高亮**：选中某个父级 Group 时，其下属覆盖的所有底层图形（Atoms）会同步呈现绿色高亮边框，帮助用户明确语义边界。
    
    ### 4.4 防版式破坏机制 (Zoom & Layout)
    
    系统提供视口缩放功能（50% - 200%）。为了解决放大后换行导致“跨行分组框（Group Fragment）”错位或断裂的问题：
    
    - **DOM 结构**：按行容器（`row-block`）构建，内部剥离为图形列（`atomic-strip`）和层级列（`levels-area`）。
    - **CSS 策略**：强制容器内 `flex-wrap: nowrap` 且 `width: max-content`，行容器外层允许横向滚动（`overflow-x: auto`）。这保证了图形与其上方的分组框在任何缩放比例下都享有严格统一的 X 轴坐标系。
    
    ### 5. 数据导入、导出与安全校验
    
    ### 5.1 数据导出 (Export)
    
    导出标准化的 JSON 文件。为保证数据的可追溯性，导出结构包含：
    
    - `session`: 操作者与页码范围。
    - `config_snapshot`: 导出瞬间的 `config.js` **配置快照**。
    - `groups` & `annotations`: 用户产生的数据。
    
    ### 5.2 数据导入与冲突阻断 (Import)
    
    导入时执行严格的数据校验，若出现异常则弹出 Confirm 警告窗供用户决策：
    
    1. **配置一致性校验**：比对当前系统 `configRoot.tasks` 与导入文件的 `config_snapshot.tasks`。如果不一致，提示存在配置差异风险。
    2. **工作区偏移修正**：比对当前工作范围与导入文件的工作范围。若不一致，系统在用户确认后将**自动接管跳转**到导入文件所限定的起止页，以确保导入的坐标数据能够被正确渲染。
    
    ### 6. 存储与状态管理
    
    - 使用浏览器 `localStorage` 进行全状态自动缓存。
    - 缓存键值：当前版本升级至 `dishu_group_label_session_v5`，从而自动清理并隔离早期版本不兼容的冗余缓存结构。
    - 右下角全局悬浮进度面板 (`progressOverlay`) 根据当前模式实时聚合计算覆盖率、层级总数或任务标记比例。
