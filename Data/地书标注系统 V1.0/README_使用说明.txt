《地书》图形语言多级分组与多任务标注系统简要使用说明

————————————————————————————————————————————————————————————————————————————————————————————————
一、系统用途

本系统用于对《地书》自动切分后的图形序列进行两类操作：

1. Grouping（分组）
2. Labeling（标注）

可对最小图形单元（Atom）及其组合形成的组（Group）进行整理和标注，用于研究《地书》与自然语言之间的联系和差异。

————————————————————————————————————————————————————————————————————————————————————————————————
二、主要文件

- index.html：主页面
- style.css：界面样式
- app.js：核心逻辑
- data/segments_manifest.js：图形单元数据
- data/config.js：标注任务配置
- images/...：图形图片

————————————————————————————————————————————————————————————————————————————————————————————————
三、启动方式

直接双击 index.html 打开。

————————————————————————————————————————————————————————————————————————————————————————————————
四、进入系统

打开页面后，需要填写：

- 学号
- 姓名
- 起始页
- 结束页

注意：
系统将“页码 + 左右页（L/R）”视为独立页面。

填写后可选择：

- 进入系统
- 恢复本地缓存

————————————————————————————————————————————————————————————————————————————————————————————————
五、两种工作模式

1. Grouping（分组）
用于将连续图形单元组合成组，并可形成多级嵌套结构。

基本操作：
- 拖拽或点击选择连续对象
- 选中后使用弹出的 Group 按钮建组
- 顶层组可通过“×”按钮解除分组

2. Labeling（标注）
用于给 Atom 或 Group 填写标注值。

基本操作：
- 切换到 Labeling 模式
- 选择一个标注任务
- 点击对象
- 在右侧面板输入或选择内容

系统通常会自动保存，不需要额外点击保存按钮。

————————————————————————————————————————————————————————————————————————————————————————————————
六、支持的标注任务类型

系统支持以下任务类型：

- text：自由文本
- single：单选
- multi：多选
- multi_text：多文本标签
- scale：量表评分

具体任务内容请参考单独的配置说明文件。

————————————————————————————————————————————————————————————————————————————————————————————————
七、对象类型

系统中的可操作对象有两类：

1. Atom：最小图形单元
2. Group：由多个 Atom 或更低层 Group 组成的组合对象

不同任务可能适用于：
- Atom 和 Group
- 仅 Group

————————————————————————————————————————————————————————————————————————————————————————————————
八、自动保存与缓存

系统会自动将当前状态保存在浏览器本地缓存中，包括：

- 用户信息
- 工作页范围
- 当前模式
- 分组结果
- 标注结果

可通过“恢复本地缓存”继续之前的工作。

————————————————————————————————————————————————————————————————————————————————————————————————
九、导出与导入

1. 导出
点击“导出数据 (JSON)”可保存当前结果。

导出内容通常包括：
- session
- groups
- annotations
- config_snapshot

2. 导入
点击“导入数据”可恢复此前导出的 JSON 文件。

————————————————————————————————————————————————————————————————————————————————————————————————
十、推荐使用流程

建议按以下顺序操作：

1. 先完成 Grouping
2. 再进行 Labeling
3. 定期导出 JSON 备份
4. 正式实验前先做小规模测试

————————————————————————————————————————————————————————————————————————————————————————————————
十一、常见问题

1. 图片不显示
请检查图片路径、images 文件夹位置，以及是否使用本地服务器运行。

2. 分组按钮不出现
请确认当前处于 Grouping 模式，并且选择的是连续对象。

3. 标注后看起来没有保存
系统一般自动保存，可检查右侧面板和对象摘要区是否已更新。

4. 导入后显示异常
请确认导入数据与当前任务配置版本一致。

————————————————————————————————————————————————————————————————————————————————————————————————
十二、说明

本系统的目标，是为《地书》建立一个可与自然语言研究框架对接的结构化标注平台。

通过分组与多层标注，可以逐步比较：

- 图形与词汇的对应关系
- 图形组合与语言结构的相似性
- 图形系统在象似性、歧义度、组合性等方面的特点
